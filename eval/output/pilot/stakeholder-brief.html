Bottom line up front

skwad doesn't just write longer reviews — it finds more real issues and makes fewer unsupported claims. When an independent, out-of-family judge opened each repo and checked the reviews line by line, skwad's claims held up 70.9% of the time versus 60.2% for our current CI reviewer, and skwad won all 8 head-to-heads.

Recommendation: greenlight skwad as our CI reviewer via a shadow rollout now, and run a 30-PR confirmatory study in parallel to lock the per-criterion picture before it fully replaces the incumbent.

How we know — four questions, answered
01

Designed to be fair

The deck was built so neither system could win on anything but merit.

  • HypothesisPre-registered primary endpoint — total score on a fixed rubric — chosen before looking at results, so we can't cherry-pick the metric that flatters skwad.
  • Rubric7 criteria × 0–3 (21-point scale): issue detection, actionability, severity accuracy, coverage, signal-to-noise, depth, novel findings. Wide scale avoids ceiling ties.
  • Neutral judgeScored by gpt-5.4 — a different model family from both reviewers (both Claude-based). No "home team" can be favored by self-preference.
  • BlindingReviews presented as anonymous "A / B", counterbalanced (which system is A is swapped run to run) plus a seeded-random run.
  • SampleReal PRs across 3 repos (frontend-mos, watson, dash-api) and 3 difficulties (easy/medium/hard); selection criteria disclosed.
02

Carried out cleanly

Same task, same diff, both systems — then scored repeatedly to kill noise.

  • Identical inputBoth reviewers got the exact same PR diff at a pinned commit SHA. No system saw extra context.
  • Repeated scoringEach PR judged 3 times and combined by median vote per criterion — one fluky run can't swing the result.
  • Position-bias checkWe tested whether slot A/B placement mattered. For skwad it didn't (p = 0.77) — the win isn't a layout artifact.
  • ReproducibleRubric, judge persona, and commit SHAs version-pinned in a manifest. The run can be re-executed.
03

Verified, not vibes

The judge didn't just read the reviews — it fact-checked them against the codebase.

  • GroundingFor each claim, the judge ran real tool calls (open file, grep) on the repo and labelled it verified / contradicted / unverified / non-falsifiable.
  • Resultskwad's claims verified 70.9% of the time vs 60.2% for CI. CI also produced contradicted claims — confidently wrong statements the code disproves.
  • CoverageJudge reasoning was 80–100% grounded in cited code, so scores reflect substance, not prose.
  • Why it mattersThis neutralizes the "skwad just writes more" worry — it wins on findings that are actually true.
04

Dependable by the numbers

A full statistical spine — including a self-imposed honesty gate.

  • SignificancePaired Wilcoxon p = 0.0078 — under 1% chance of seeing this if the systems were truly equal.
  • Effect sizeCliff's δ = 0.78 (large), 95% BCa bootstrap CI [0.50, 1.00] — the interval excludes "no difference".
  • Reliability gateWe measured judge agreement (Krippendorff's α) and excluded the 3 criteria too subjective to score consistently. The gate worked.
  • No p-hackingExploratory breakdowns corrected with Benjamini–Hochberg, and we report honestly that they're not yet individually conclusive at N=8.

What this pilot can — and can't — claim yet

Straight talk, because a result you can defend beats one you can't:

  • The headline is solid. skwad scoring higher overall is significant and large, cross-model and code-verified — direction is robust.
  • The per-criterion breakdown isn't conclusive yet. At N=8, after honest multiple-comparison correction, individual criteria don't reach significance on their own. We need more PRs to say where exactly skwad wins.
  • Magnitude may be slightly generous. A GPT judge tends to reward structure, and skwad's output is more structured — so the true margin could be a touch smaller (the verification-rate gap, which is style-immune, guards against this).
  • Sample was curated. PRs were hand-picked from our repos; the confirmatory run should sample more broadly.
